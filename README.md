Reporte de incidencias ( residente del local) 

 Una aplicación web que permita a los residentes reportar incidencias relacionadas con el edificio en el que viven. Estas incidencias deben incluir detalles como la ubicación exacta (bloque o inmueble), tipo de problema, descripción y cualquier información relevante. Además, la aplicación debe permitir al equipo administrativo recibir notificaciones sobre las incidencias reportadas, visualizarlas y marcarlas como resueltas una vez se hayan solucionado.

✅ Control de manejo de la pagina se caracterizan por :
* Cliente, 
* Personal o Mantenimiento del area,
* Administrador

✅ Requerimientos funcionales.

* CRUD (Crear, Leer, Actualizar, Eliminar) de incidencias para los residentes.
    * Los incidentes deben tener datos básicos cómo:
        * Asunto, tipo de incidente, detalle de incidentes, imagen, estado.
* Filtrado y búsqueda de incidencias por diferentes criterios (estado, fecha).

Registro
Todas las incidencias del servicio deben ser registradas, y cada incidencia debe registrarse de forma independiente. La información a registrar generalmente incluye:
* Identificador único.
* Categorización.
* Urgencia, impacto y prioridad.
* Fecha y hora.
* Persona/grupo que registra la incidencia.
* Canal de entrada.
* Datos del usuario.
* Síntomas.
* Estado.
* CIs (Configuration Items, elementos de configuración) asociados.
* Persona/grupo asignado para la resolución.
* Problema/Known error asociado.
* Actividades realizadas para la resolución.
* Fecha y hora de la resolución.
* Categoría del cierre.
* Fecha y hora de cierre.

La prioridad de la incidencia suele depender de:
* La urgencia: rapidez con que la incidencia necesita ser resuelta.
* El impacto: generalmente se determina por el número de usuarios afectados.
